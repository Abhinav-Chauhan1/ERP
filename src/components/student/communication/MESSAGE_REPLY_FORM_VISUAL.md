## Color Scheme

### Role Badges

- **ADMIN**: Purple background (`bg-purple-100 text-purple-700 border-purple-200`)
- **TEACHER**: Blue background (`bg-blue-100 text-blue-700 border-blue-200`)
- **STUDENT**: Green background (`bg-green-100 text-green-700 border-green-200`)

### Component Colors

- **Card Border**: Primary color with 20% opacity (`border-primary/20`)
- **Original Message Section**: Muted background (`bg-muted/50`)
- **Avatar Fallback**: Blue background (`bg-blue-100 text-blue-700`)
- **Error Text**: Red (`text-red-500`)
- **Muted Text**: Gray (`text-muted-foreground`)

## Best Practices

1. **Always provide onCancel**: Users need a way to exit
2. **Handle onReply callback**: Refresh data after successful reply
3. **Show loading states**: Prevent duplicate submissions
4. **Preserve context**: Original message always visible
5. **Test on mobile**: Ensure touch targets are adequate
6. **Validate input**: Client-side and server-side validation
7. **Handle errors gracefully**: Show user-friendly error messages
8. **Provide feedback**: Success/error toast notifications
